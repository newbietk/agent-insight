# 上下文追踪页面设计 (Context Tab)

## 1. 页面定位

新增 Tab: **上下文**

```
[Overview] [Turns] [Workflow ✦] [Subagents] [Skills] [Interactions] [Workflow (AI)] [上下文 📊]
```

核心概念：
- **独立上下文**: 主 agent (build) 和每个 subagent 维护各自独立的对话历史
- **上下文增长**: 随着对话进行，input_tokens 逐步累积
- **缓存机制**: cache_read 表示已缓存的 token，减少重复计算
- **上下文窗口**: 模型有最大 token 限制（如 128K、200K），接近时会触发压缩

## 3. 数据来源

### 3.1 数据提取 SQL

从 opencode 数据库的 `message` 表提取：

```sql
SELECT 
  m.id,
  m.time_created,
  json_extract(m.data, '$.agent') as agent,
  json_extract(m.data, '$.tokens.input') as input_tokens,
  json_extract(m.data, '$.tokens.cache.read') as cache_read,
  json_extract(m.data, '$.tokens.output') as output_tokens,
  json_extract(m.data, '$.tokens.total') as total_tokens,
  json_extract(m.data, '$.role') as role
FROM message m
WHERE m.session_id = ?
  AND json_extract(m.data, '$.role') = 'assistant'
ORDER BY m.time_created
```

### 3.2 计算逻辑

1. **按 agent 分组**: 每个 agent 独立计算上下文增长
2. **上下文大小**: `input_tokens` 表示该轮次的上下文大小
3. **缓存命中率**: `cache_read / (cache_read + input_tokens)`
4. **峰值检测**: 找出每个 agent 的最大 `input_tokens`
5. **事件识别**:
   - 开始: 某个 agent 的第一条消息
   - 增长: `input_tokens` 显著增加（如 >5K）
   - 峰值: 达到最大值
   - 警告: 接近模型限制（如 >80% 的 128K）

## 4. API 设计

### GET /api/observe/context-growth?taskId=xxx

```typescript
interface ContextGrowthResponse {
  agents: {
    [agentName: string]: {
      isRoot: boolean;
      turns: Array<{
        turnIndex: number;
        timestamp: string;
        inputTokens: number;
        cacheRead: number;
        outputTokens: number;
        totalTokens: number;
      }>;
      stats: {
        peak: number;
        average: number;
        min: number;
        totalTurns: number;
        avgGrowthPerTurn: number;
        cacheHitRate: number;
        cacheSavedTokens: number;
      };
      events: Array<{
        turnIndex: number;
        timestamp: string;
        type: 'start' | 'growth' | 'peak' | 'warning' | 'end';
        contextSize: number;
        note?: string;
      }>;
    };
  };
  insights: string[];
}
```

## 7. 页面价值

1. **性能优化**: 识别上下文过大的 agent，优化 prompt 或拆分任务
2. **成本分析**: 理解缓存效率，优化 token 消耗
3. **调试工具**: 追踪上下文异常增长的原因
4. **对比分析**: 不同 agent 的上下文管理策略对比
5. **预警机制**: 提前发现接近模型限制的 agent
